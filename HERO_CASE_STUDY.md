# Hero Case Study — SAF-T PT / E-invoicing Traceability

This repository now uses **one deep hero case** instead of several shallow demo examples:

> **Compliance-to-Product Traceability for SAF-T PT / E-invoicing**  
> Transform a realistic source excerpt into obligations, reviewer corrections, requirements, Jira payloads, QA coverage, safer release communication, incident learning and a signed audit report.

Read the full case here:

- [`docs/HERO_SAF_T_EINVOICING_CASE.md`](docs/HERO_SAF_T_EINVOICING_CASE.md)

## Why this case exists

A top-tier portfolio cannot rely only on generic AI outputs. It needs one workflow that proves product judgment:

```text
input document
→ extracted obligations
→ reviewer corrections
→ before/after requirement
→ Jira ticket
→ QA case
→ risky release note
→ safer release note
→ incident if missed
→ final audit report
```

## Portfolio thesis demonstrated

This case shows that the product is not “an AI generator.” It is an **audit-aware human-in-the-loop product workflow** for regulated enterprise software.

## What is synthetic

The source excerpt, metrics and incident are synthetic but realistic. They are designed to demonstrate the workflow safely without claiming legal accuracy or exposing customer data.
