From 15 November 2026, SEPA and SWIFT CBPR+ payment messages should no longer rely on fully unstructured postal addresses. Payment address data should use either a hybrid format, where Town Name and Country Code are provided in structured fields plus up to two free-text address lines, or a fully structured format where Street Name, Building Number, Post Code, Town Name and Country Code are represented as dedicated fields. The workflow should flag unclear handling of legacy counterparty data, non-Latin characters and migration of existing customer/beneficiary address records for human review.
