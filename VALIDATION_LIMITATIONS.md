# Validation and Limitations — v16

This project is intentionally presented as a **portfolio-grade prototype**, not as a production compliance product.

## What is verified by code

- Source IDs cited in generated claims must exist in the retrieved chunk set.
- Model-proposed `quote_support` values are checked against the cited source text.
- Generated claims are compared against cited source chunks using deterministic support checks.
- Weak, missing or unsupported citations are flagged for human review.
- Each extracted obligation can be linked to a best supporting quote.
- Version changes can be compared with a rule-based obligation diff.
- Output completeness is measured against expected structured sections.
- Risky wording is detected with explicit rule patterns.
- A synthetic obligation benchmark compares generated obligations with a hand-written gold set.
- Five Interpret Quality Metrics are calculated from the benchmark and rule-based citation rows: obligation precision, obligation recall, citation support rate, unsupported claim rate and human review rate.

## What is model-generated

- Obligations, requirements, release notes, summaries and risk descriptions are generated by the language model.
- Any model confidence label is advisory only.
- Model-generated claims must not be treated as compliance determinations.

## What requires human review

- Legal meaning of regulatory text.
- Final compliance interpretation.
- Production implementation decisions.
- Customer-facing wording in regulated contexts.
- Any operational use involving private or sensitive data.

## Benchmark status

The repository includes two synthetic benchmarks:

```bash
python scripts/run_interpret_benchmark.py
python scripts/run_interpret_obligation_benchmark.py
python scripts/run_interpret_quality_metrics.py
```

The first measures retrieval coverage. The second compares generated obligations against a hand-written synthetic gold set using obligation precision/recall/F1 proxies. The app also combines these results with rule-based citation checks to show citation support rate, unsupported claim rate and human review rate.

## Known limitations

- Citation support checking is a rule-based support check, not legal proof.
- Quote-level grounding checks whether text support exists, not whether the interpretation is legally complete.
- Demo Mode uses deterministic local embeddings for zero-cost testing.
- API Mode uses OpenAI embeddings for stronger retrieval.
- The benchmark dataset is synthetic and small.
- No authentication, RBAC or enterprise document security is implemented in this portfolio demo.
- Audit logs are exportable but not persisted in an enterprise-grade audit store.
- Screenshots in `assets/screenshots` are mockups until replaced with live Streamlit screenshots.

## v16 metrics limitation

Obligation precision and recall require an independent gold-standard obligation set for the same document. Without that file, the application deliberately displays these metrics as **N/A**.

The citation-related metrics are run-specific, but still rule-based. They should be used as review signals, not compliance proof.
