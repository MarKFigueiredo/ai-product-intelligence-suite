# Demo Script: AI Product Intelligence Suite 1.04

## One-line opener

“In 60 seconds, this app shows how a regulated product team can convert ambiguous compliance input into reviewed requirements, QA coverage and safer release communication.”

---

## 60-second version

### 0–10 seconds — Positioning

“This is a portfolio case study, not a commercial product. It demonstrates how I design AI product workflows for regulated enterprise software.”

**Click:** Start Here — Hero Demo

### 10–25 seconds — Source to obligations

“The hero case is SAF-T PT / e-invoicing. We start with compliance-style source material and extract obligations with source evidence.”

**Show:** source text + extracted obligations

### 25–40 seconds — Human review to product artifacts

“The key product decision is not to trust AI blindly. A reviewer marks obligations as correct, weak or missing. Reviewed obligations become requirements, Jira-style tickets and QA cases.”

**Show:** reviewer correction + before/after requirement

### 40–55 seconds — QA and release risk

“Every obligation that enters the release pipeline needs QA coverage, including negative tests. The release note is scanned for risky claims and rewritten with safer scope.”

**Show:** negative coverage + risky/safer release note

### 55–60 seconds — Close

“The final output is an audit-ready package showing what was supported, weak, missing and reviewed.”

**Close line:** “This is the kind of AI product workflow I would design for regulated enterprise teams.”

## 90-second version

### 0–10 seconds — Hook

“In regulated enterprise software, the hard part is not generating text. The hard part is preserving traceability from compliance input to product requirement, QA coverage, release wording and audit evidence.”

### 10–20 seconds — Project positioning

“This project is a portfolio case study, not a commercial product. It shows how I think about AI product workflows, human review and enterprise readiness.”

### 20–35 seconds — Interpret

“Here is the source material. The app extracts candidate obligations and keeps evidence linked to the original input.”

### 35–50 seconds — Review

“The reviewer mode is the safety layer. Obligations can be marked correct, incorrect or weak. Corrections are saved as product evidence, not hidden in chat history.”

### 50–65 seconds — Product and QA

“Reviewed obligations become requirements, Jira-style tickets and QA cases. I added mandatory negative test coverage because regulated workflows fail when teams only test the happy path.”

### 65–80 seconds — Release communication

“The release note scanner catches risky claims like ‘guarantees compliance’ and rewrites them with scope, caveats and human-review boundaries.”

### 80–90 seconds — Audit and hiring signal

“The final package shows supported, weak, missing and reviewed artifacts. What this demonstrates is domain understanding, AI workflow design, risk-aware communication, QA discipline and enterprise readiness judgment.”

## Demo do-not-say list

Avoid:

- “This guarantees compliance.”
- “This is production-ready.”
- “This replaces legal/compliance review.”
- “This fully automates regulated product work.”

Use instead:

- “This supports review workflows.”
- “This makes evidence and gaps visible.”
- “This is a portfolio prototype.”
- “Human review remains required.”
