# Real usage metrics and data portability

v1.04 separates **synthetic evaluation** from **real usage telemetry**.

Synthetic benchmark files are still useful for a portfolio review, but the app now also records metrics that are created when someone actually uses the product workflows.

## What is now real and persisted locally

When a workflow is generated in the app, v1.04 writes a row to local SQLite:

```text
.local_runs/usage_metrics.db
```

The app persists:

- workflow name;
- actor and role simulation;
- model and demo/API mode;
- input hash, not source text by default;
- output hash;
- input/output byte size;
- source count;
- generation latency;
- derived quality/product metrics;
- signed row metadata.

It also records import/export/connector events in the same local usage database.

## Tables

| Table | Purpose |
|---|---|
| `workflow_runs` | One row per actual generated workflow run. |
| `metric_snapshots` | Numeric metrics derived from each run. |
| `data_events` | Imports, exports, connector outbox saves and optional live connector calls. |

## Import/export

The new **Usage Metrics & Data** page can export:

- complete usage JSON;
- ZIP package containing JSON + CSV tables;
- workflow runs CSV;
- metric snapshots CSV;
- data events CSV.

It can also import a previously exported usage JSON file.

This means the portfolio can start empty, collect usage over time, then export the real evidence produced by reviewers or interview demos.

## Local export packages

Each workflow now includes a **Save structured export locally** action. This stores a signed local JSON package in:

```text
.local_exports/
```

This is separate from browser downloads. Browser downloads are useful for the reviewer, but the app cannot reliably know whether a browser download was completed. Local export saves are therefore tracked explicitly as durable data events.

## Connectors

v1.04 adds a safer connector model:

1. **Local outbox mode** — default. The app saves real Jira/GitHub/Slack-shaped payload files locally in:

```text
.local_connector_outbox/
```

2. **Live connector mode** — optional. If environment variables are configured and the user explicitly enables live calls, the app can attempt real HTTP calls to Jira/GitHub/Slack.

The app never sends live connector requests by default.

## Why this matters for hiring

This shows product judgment beyond demo screens:

- metrics are generated by use, not just written in documentation;
- outputs can be imported/exported;
- audit and data events are persisted;
- connectors have a real local handoff and optional live path;
- sensitive source text is not silently stored by default;
- synthetic benchmarks and real usage metrics are clearly separated.


## v1.04 QA quality gate

v1.04 persists mandatory negative test coverage metrics for workflows that contain obligations and QA rows. These metrics are generated from actual run outputs and make it possible to track whether the portfolio app is producing release-ready QA evidence or only happy-path coverage.
